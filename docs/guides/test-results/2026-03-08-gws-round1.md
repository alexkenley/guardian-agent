## Notes

- All 5 write approval gates passed: Calendar, Drive, Docs, Sheets, Gmail send
- All 5 denial decisions accepted correctly
- 2 FAILs were false negatives: the tool executed (bypassed approval correctly) but status was `failed` instead of `error` — harness didn't accept `failed` as a valid non-approval status. **Fixed** in subsequent commit.
- CLI not resolving from project `node_modules` because `@googleworkspace/cli` was not a project dependency. **Fixed**: added to `package.json` dependencies.
- Read/discovery tests skipped because CLI wasn't available to the subprocess.
